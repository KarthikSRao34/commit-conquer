## 8. APPLICATION STARTUP SEQUENCE

### Exact startup steps from fresh clone

```bash
# 1. Install dependencies (root and all workspaces)
npm install

# 2. Set up backend environment
cp packages/server/.env.example packages/server/.env  # (or create it manually)

# 3. No database setup needed — data is in-memory

# 4. No seed script needed — seed data is baked into services

# 5. Start backend + frontend together
cd apps/storefront
npm run start

# OR run them separately:
# Terminal 1:
npm run server:dev        # from apps/storefront/ or root

# Terminal 2:
npm run dev               # from apps/storefront/
```

### Dependency between frontend and backend
- Frontend build (`npm run build`) is **independent** — compiles TypeScript/JSX to static files
- Frontend dev server proxies `/api` requests to backend (see `apps/storefront/vite.config.ts`)
- Frontend requires backend running at localhost:4000 to serve API requests, but build succeeds without it

### Build time
Not measured; typical Vite build: 5-15 seconds

### Backend warm-up
No warm-up period — starts immediately and listens on port 4000

- ✅ **CRITICAL:** `apps/storefront/vite.config.ts` line 11: proxy `target: "http://localhost:4000"`
- ✅ **IMPACT:** Pipeline running in Docker/CI will fail because frontend can't reach backend — need to parameterize URLs

### Unusual repo requirements
- ✅ **IMPORTANT:** All data is in-memory (no database) — data is lost on restart. Seed data is rebuilt on each server start.
- ✅ **IMPORTANT:** Migrations are empty shells — they don't run anything
- ✅ **IMPORTANT:** No environment variables strictly required for startup (all have defaults), but hardcoded localhost URLs will break in CI
- ✅ **IMPORTANT:** Admin routes require `X-Admin-Secret` header — this is development-mode only (bypassed when `NODE_ENV !== production`)
- ✅ **IMPORTANT:** Stripe provider is an empty file — payment processing is stubbed

### Summary for pipeline setup
- **Backend:** Single `ts-node` command; no build step required
- **Frontend:** Standard Vite build (`vite build` → `dist/`)
- **Both:** Can be tested independently or together
- **Main blocker:** Hardcoded localhost URLs need environment parameterization
- **In-memory data:** Tests start with consistent seed state
- **External services:** No database, migrations, or external service calls required (optional Stripe is stubbed)
